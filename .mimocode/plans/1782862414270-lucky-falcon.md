# Plano: compose:route + compose:parallel

## Objetivo

Tornar o compose agent um orquestrador completo que delega **todas** as operações (buscas, leituras, validações, escritas) para subagentes, usando `route` para escolher o agente certo e `parallel` para executar tarefas independentes em paralelo.

## Contexto

O projeto já tem:
- 3 tipos de subagentes: `explore` (leitura), `general-medium` (implementação), `general-heavy` (arquitetura)
- Skills compose existentes (brainstorm, plan, tdd, debug, verify, etc.)
- A skill `route` já existe em `skills/compose/route/SKILL.md` mas é básica
- A skill `parallel` já existe em `skills/compose/parallel/SKILL.md` mas é genérica

O compose agent precisa de skills de roteamento e paralelismo integradas ao seu workflow de orquestração.

---

## Mudanças

### 1. Atualizar `skills/compose/route/SKILL.md`

**Arquivo:** `skills/compose/route/SKILL.md`

Reescrever a skill para ser o núcleo do roteamento de delegação:

- **Matriz de decisão expandida**: mapear tipos de operação (read, write, search, validate, implement, debug, analyze) para o agente ideal
- **Regras de routing claras**: quando usar explore vs general-medium vs general-heavy
- **Padrão de delegação**: como construir o prompt do subagente com contexto adequado
- **Custo-benefício**: sempre usar o agente mais barato que consegue lidar com a tarefa
- **Exemplos práticos**: tabela com operações reais e qual agente usar

Estrutura pretendida:
```
## Decision Matrix (expandida)
| Operação | Agente | Custo | Razão |

## Routing Rules
1. Default to cheapest
2. Escalate only when needed
3. Never ask user which agent

## Delegation Pattern
- Como construir o prompt
- Como passar contexto
- Formato de retorno esperado

## Examples
- Read file → explore
- Find usages → explore
- Implement feature → general-medium
- Fix bug → general-medium
- Design architecture → general-heavy
- Optimize performance → general-heavy
- Validate test suite → explore (read) + general-medium (fix)
```

### 2. Atualizar `skills/compose/parallel/SKILL.md`

**Arquivo:** `skills/compose/parallel/SKILL.md`

Reescrever para suportar dispatch paralelo de subagentes com o pattern do compose:

- **Quando paralelizar**: tarefas independentes, sem estado compartilhado
- **Como dispatchar**: usar `actor` tool com múltiplas chamadas na mesma mensagem
- **Padrão de prompt**: cada subagente recebe escopo focado + contexto isolado
- **Integração com route**: cada tarefa paralela passa por route para escolher o agente certo
- **Review pós-paralelo**: como integrar resultados de múltiplos subagentes

Estrutura pretendida:
```
## Overview
Delegar tarefas independentes para subagentes com contexto isolado.

## When to Use
- 2+ tarefas independentes
- Sem estado compartilhado
- Sem dependências sequenciais

## The Pattern
1. Identificar domínios independentes
2. Para cada domínio: route → escolher agente → construir prompt
3. Dispatch paralelo (múltiplas chamadas actor na mesma mensagem)
4. Aguardar resultados
5. Revisar e integrar

## Integration with Route
Cada tarefa paralela usa route para escolher o agente:
- Leitura → explore
- Implementação → general-medium
- Análise → general-heavy

## Prompt Structure
- Focado: um problema claro
- Auto-contido: todo contexto necessário
- Específico sobre output: o que retornar

## Common Mistakes
- Tarefa ampla demais
- Sem contexto
- Sem restrições
- Output vago
```

### 3. Atualizar `prompts/compose-system.txt`

**Arquivo:** `prompts/compose-system.txt`

Adicionar instruções sobre o padrão de delegação:

```
You are the Compose Agent — an orchestrator that coordinates specialized skills into coherent workflows.
When a skill matches your task, invoke it via the skill tool.
Follow each skill's guidance exactly. Never skip skill invocation.
Every decision goes through the question tool — never stop with natural language questions.
Route every decision through the compose:ask skill.
Use the skill tool to load skills. When you invoke a skill, its content is loaded — follow it directly.

## Delegation Pattern
You are an orchestrator, not an implementer. Delegate ALL operations to subagents:
- Use compose:route to choose the right agent for each task
- Use compose:parallel for independent tasks that can run concurrently
- Never read files, grep, or edit code directly — delegate to explore or general agents
- Always provide focused context to subagents (they don't see your history)
```
