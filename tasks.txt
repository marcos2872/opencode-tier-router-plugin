# Tarefa futura de implementação — T34

**Tarefa**: Injetar o protocolo de subagente no prompt de sistema do subagente.

**Req**: ALIGN-34

**Status**: Pendente — implementar em uma tarefa futura.

## Objetivo

Quando o plugin rotear uma solicitação para um subagente, adicionar esta regra ao prompt enviado ao subagente:

```md
Do not dispatch sub-sub-agents. Do not ask the user questions unless blocked.
```

O objetivo é fazer o subagente se comportar como um subagente nativo do OpenCode: executar o trabalho atribuído diretamente e evitar delegação recursiva.

## Onde implementar

- `src/prompts.ts`: criar um builder de prompt, por exemplo `buildSubagentProtocol()`.
- `src/plugin-orchestrator.ts`: em `handleSystemTransform`, se `input.sessionID` identificar uma sessão de subagente, adicionar `buildSubagentProtocol()` ao `output.system` e retornar antes dos blocos de hard-block/routing do router.

## Comportamento esperado

- O prompt do subagente deve **adicionar** esta regra ao prompt existente. Ele não deve substituir o prompt inteiro.
- Subagentes devem manter suas permissões atuais de ferramentas.
- Subagentes devem manter `task` negado por `permission.ask` e `event`.
- Sessões principais não devem receber este bloco exclusivo de subagente.
- Sessões principais em hard-block devem continuar recebendo o prompt de router/hard-block existente.

## Formato mínimo de implementação

```ts
if (input.sessionID && this.subagentSessions.has(input.sessionID)) {
  output.system = output.system ?? [];
  output.system.push(buildSubagentProtocol());
  return;
}
```

## Cobertura de testes

Adicionar/ajustar testes unitários em `test/index.test.ts` para verificar:

- A saída de `system.transform` de um subagente inclui `Do not dispatch sub-sub-agents. Do not ask the user questions unless blocked.`
- O bloco de subagente é adicionado, sem substituir o prompt inteiro.
- Uma sessão principal de router não recebe o bloco exclusivo de subagente.
- O comportamento existente de hard-block/routing continua intacto.
